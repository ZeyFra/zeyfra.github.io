ArrayList是Java中最常用的集合之一，它继承了抽象类AbstractList，同时还实现了Serializable、Cloneable、RandomAccess接口，是一个支持快速访问、复制、序列化的集合。

AbstractList

AbstractList 是一个抽象类，实现了List 接口，是Java集合框架AbstractCollection中的一个子接口

List

List是一个有序集合接口，实现该接口，可以精确控制列表中每个元素的插入位置，可以通过整数索引访问元素，并搜索列表中的元素，List允许元素重复，而Set接口不允许

RandomAccess

  • RandomAccess接口是一个标记型接口（Marker），只要List集合实现这个接口，就能支持快速随机访问，可以提高查找效率，实现RandomAccess接口的List集合采用一般的for循环遍历，而未实现这接口则采用迭代器interator。
  • LinkedList中并未实现RandomAccess接口。
  • 而ArrayList用for循环遍历比用iterator迭代器遍历快，LinkedList用iterator迭代器遍历比for循环遍历快，所以List集合的不同子类应当采用不同的遍历方式，以便能够提高性能。
  • 关于RandomAccess接口，更多详情可参考 这篇文章。

Cloneable

  • Cloneable是标记型的接口，实现 Cloneable来表示该对象能被克隆，能使用clone()方法。
  • 如果没有实现Cloneable接口的类对象调用clone()就会抛出CloneNotSupportedException异常。

Serializable

  • Serializable也是一个标记型接口，一个类只有实现了Serializable接口，它的对象才能被序列化
  • 序列化是指把对象转换为字节序列的过程，我们称之为对象的序列化，就是把内存中的这些对象变成一连串的字节(bytes)数据，这样对象数据便可以二进制文本的方式被持久化到了磁盘文件中。
  • 相应的也有反序列化，与序列化含义相反，将持久化的对象字节流数据通过IO转化为Java对象

关于fast-fail

  • fail-fast 机制，即快速失败机制，是Java集合(Collection)中的一种错误检测机制。

  • 当在迭代集合的过程中该集合在结构上发生改变的时候，就有可能会发生fail-fast，即抛出 ConcurrentModificationException异常。fail-fast机制并不保证在不同步的修改下一定会抛出异常，它只是尽最大努力去抛出，所以这种机制一般仅用于检测bug

  • ArrayList底层是基于数组实现的动态容量可变的数据结构

  • ArrayList的扩容机制默认是扩大为原始容量的1.5倍

  • ArrayList实现了RandomAcess接口，支持高效的快速随机访问，查找效率高

  • ArrayList对于增删操作效率较低，因为是基于数组的实现的数据结构，插入或者删除一个元素时需要大量的移动操作

  • ArrayList是线程不安全的类,如果你想用线程安全的,可以用Vector，或者追加sychornized修饰符

  • ArrayList需要在列尾预留一定空间
